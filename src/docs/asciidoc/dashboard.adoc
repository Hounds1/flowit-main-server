== 개요

이 문서는 대시보드 화면에서 조합해 사용할 API를 한곳에 모아 설명합니다.
각 API의 상세 요청/응답 계약은 개별 API 문서를 기준으로 확인합니다.

[cols="1,4",role="endpoint-summary"]
|===
|목적 |대시보드 첫 화면 렌더링에 필요한 API 목록과 사용 위치를 정리합니다.
|대상 화면 |워크스페이스 대시보드
|인증 |모든 API는 JWT access token이 필요합니다.
|권한 |요청자는 해당 워크스페이스의 활성 멤버여야 합니다.
|시간 값 |서버에서 유통하는 시간 값은 Long 타입의 Unix epoch seconds 기준입니다.
|주의 |이 문서는 API 계약의 원본이 아니라 화면 조립 가이드입니다.
|===

== 사용 API

[cols="5%,22%,8%,40%,25%",options="header",role="api-table dashboard-api-table"]
|===
|순서 |API |Method |Path |대시보드 사용 위치

|1
|link:workspaces-tasks-indicators.html[작업 인디케이터 조회 API]
|``GET``
|``/v1/workspaces/{workspaceId}/tasks/indicators``
|전체 업무, 진행 중, 오늘 마감, 만료 인디케이터 카드에 사용합니다.

|2
|link:workspaces-activity-records.html[워크스페이스 최근 활동 API]
|``GET``
|``/v1/workspaces/{workspaceId}/activity-records``
|최근 활동 목록 또는 활동 피드에 사용합니다.

|3
|link:workspaces-tasks-list.html[작업 목록 조회 API]
|``GET``
|``/v1/workspaces/{workspaceId}/tasks``
|대시보드의 작업 목록, 진행 중 작업 목록, 필터링된 업무 리스트에 사용합니다.
|===

== 권장 조립 방식

대시보드 초기 렌더링에서는 세 API를 서로 독립적으로 호출할 수 있습니다.
각 응답은 서로의 결과를 필요로 하지 않으므로 클라이언트에서 병렬 호출해도 됩니다.

[cols="1,4",options="header",role="api-table"]
|===
|영역 |권장 사용 방식

|인디케이터
|작업 인디케이터 조회 API를 사용합니다. 만료는 마감 예정 시각이 지났고 완료되지 않은 업무 수입니다.

|최근 활동
|워크스페이스 최근 활동 API를 사용합니다. 서버가 완성 문장을 내려주지 않으므로 ``actor``, ``target``, ``action``, ``changes``를 화면 정책에 맞게 조립합니다.

|작업 목록
|작업 목록 조회 API를 사용합니다. 필요한 화면에 따라 ``mine``, ``status``, ``assigneeMemberId``, ``tag``, ``keyword``, ``dueFrom``, ``dueTo``, ``page``, ``size``를 조합합니다.
|===

== 초기 로드 예시

[source,http]
----
GET /v1/workspaces/2001/tasks/indicators
GET /v1/workspaces/2001/activity-records?topic=ALL&rangeDays=5&page=0&size=20
GET /v1/workspaces/2001/tasks?mine=true&page=0&size=20
----

작업 목록은 화면 성격에 따라 필터 없이 전체 업무를 보여줄 수도 있고, ``mine=true``처럼 현재 사용자에게 속한 업무만 보여줄 수도 있습니다.
``mine=true``는 현재 사용자가 담당자인 작업 또는 현재 사용자가 생성한 작업을 조회합니다.
현재 사용자가 생성했고 다른 담당자가 지정된 작업도 포함합니다.
대시보드가 "내 진행 중 업무" 섹션을 갖는다면 ``mine=true``와 ``status=IN_PROGRESS``를 함께 사용하는 편이 명확합니다.
