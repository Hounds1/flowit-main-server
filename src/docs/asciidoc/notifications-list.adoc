== 알림 목록 조회

[cols="1,4",role="endpoint-summary"]
|===
|메서드 |``GET``
|경로 |``/v1/notifications``
|API 상태 |[.badge.stability.experimental]#Experimental#
|목적 |현재 인증된 사용자의 개인 알림 목록과 읽음/확인 카운트를 조회합니다.
|인증 |JWT access token이 필요합니다.
|정렬 |최근 생성된 알림이 먼저 내려옵니다.
|페이지 |``page``는 0부터 시작합니다. ``size``는 생략 시 20, 최소 1, 최대 100으로 보정됩니다.
|읽음 상태 |``read``는 현재 사용자 기준의 읽음 여부입니다. 알림 자체가 아니라 수신자별 상태입니다.
|상태 변경 |목록 조회만으로 ``read`` 또는 ``seen`` 상태는 변경되지 않습니다.
|소켓 연계 |접속 중에는 link:websocket-notifications.html[알림 WebSocket]으로 같은 알림 payload가 추가 전달될 수 있습니다.
|===

== 클라이언트 권장 흐름

알림 목록 조회 API는 상태를 변경하지 않습니다.

사용자가 알림 아이콘을 클릭해 오버레이를 열고 목록이 화면에 노출된 경우, 클라이언트는 목록 조회 후 link:notifications-seen.html[알림 확인 처리 API] 호출을 권장합니다.

단, 백그라운드 동기화, 선조회, 캐시 갱신 목적의 목록 조회에서는 ``/seen``을 호출하지 않아야 합니다. ``/seen``은 목록 API가 호출되었다는 사실이 아니라 사용자가 알림 목록을 실제로 확인할 수 있는 화면 상태에 진입했음을 서버에 알려주는 호출입니다.
