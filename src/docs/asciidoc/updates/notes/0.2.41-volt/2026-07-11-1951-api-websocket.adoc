Integration Note

Added:: 작업 댓글 생성 알림이 개인 알림 API와 WebSocket에 추가되었습니다.
+
* ``POST /v1/workspaces/{workspaceId}/tasks/{taskId}/comments``로 댓글이 생성되면 수신자에게 ``TASK_COMMENT_CREATED`` 알림이 생성됩니다.
* 수신자 후보는 작업 생성자와 현재 담당자이며 댓글 작성자는 제외합니다. 작업 생성자와 담당자가 같은 사용자이면 한 번만 전달합니다.
* 과거 댓글 작성자는 별도 구독자 또는 참여자로 취급하지 않으므로 수신자 후보에 포함하지 않습니다.
* 댓글 작성자만 수신자 후보로 남는 경우에는 알림을 생성하지 않습니다.

Client:: 기존 개인 알림 구독과 응답 조립 방식을 그대로 사용합니다.
+
* WebSocket 구독 destination은 기존과 동일한 ``/user/queue/notifications``입니다.
* WebSocket payload와 ``GET /v1/notifications``의 ``data.items[]``는 동일한 ``NotificationAlertResponse`` 구조를 사용합니다.
* ``type``은 ``TASK_COMMENT_CREATED``이고 ``profile.source``는 ``ACTOR``입니다. 댓글 작성자의 표시 정보는 ``actor``와 ``profile``을 사용합니다.
* ``subject.type``은 ``TASK``, ``subject.id``는 ``taskId``, ``subject.name``은 댓글 생성 시점의 작업 제목 스냅샷입니다.
* 댓글 본문은 알림 payload에 포함하지 않으며 ``changes``는 빈 배열입니다. 클라이언트는 상세 diff 없이 댓글 생성 문구를 조립합니다.
* ``link.type``은 ``TASK_DETAIL``입니다. ``link.workspaceId``와 ``subject.id``로 작업 상세 화면에 이동할 수 있습니다.
* 현재 알림 계약에는 ``commentId``나 댓글 anchor가 없으므로 특정 댓글로 직접 이동하거나 자동 스크롤할 수 없습니다.
