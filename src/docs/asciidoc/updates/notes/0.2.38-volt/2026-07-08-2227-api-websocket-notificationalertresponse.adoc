Integration Note

Changes:: 알림 목록 조회 API와 알림 WebSocket의 ``NotificationAlertResponse`` 프로필 계약이 변경되었습니다.
+
* ``actor.profileImageUrl`` 필드를 제거했습니다.
* 클라이언트는 알림의 대표 프로필 이미지를 ``actor`` 또는 ``subject``에서 직접 추론하지 않고 서버 지정 ``profile`` 객체를 사용하도록 변경되었습니다.
* ``profile`` 객체는 ``source`` 와 ``profileImageUrl`` 을 제공합니다.
* ``profile.source`` 는 ``NotificationProfileSourceType`` 값이며 ``ACTOR``, ``SUBJECT``, ``RECIPIENT`` 중 하나입니다.
* ``source`` 값은 알림 ``enum reference`` 문서에서 정확한 값을 참조해주세요.
* ``profile.profileImageUrl`` 이 ``null`` 이면 클라이언트는 ``profile.source`` 를 기준으로 사용해야 할 ``actor.name``, ``subject.name`` 또는 현재 사용자 이름의 첫 글자를 추론할 수 있습니다.
* 대표 프로필 이미지는 응답 조립 시점의 현재 프로필 이미지 조회 URL이며, 알림 발생 시점의 이미지 경로 스냅샷이 아닙니다.
* 따라서 현재 프로필 이미지가 변경되는 경우 지난 알림의 이미지 경로도 함께 변경됩니다.
* 파일 관리 정책에 따라 더 이상 워크스페이스 멤버가 아니게 된 경우 이미지 경로가 ``null``로 반환될 수 있습니다.

Carryover:: 대시보드 작업 인디케이터와 내 작업 조회 필터가 추가되었습니다.
+
* ``GET /v1/workspaces/{workspaceId}/tasks/indicators`` 작업 인디케이터 조회 API가 추가되었습니다.
* 작업 인디케이터는 ``total``, ``inProgress``, ``dueToday``, ``pendingReview`` 카운트를 제공합니다.
* ``pendingReview`` 는 리뷰 기능이 구현되기 전까지 항상 ``0`` 으로 응답합니다.
* ``GET /v1/workspaces/{workspaceId}/tasks`` 작업 목록 조회 API에 ``mine`` 쿼리 파라미터가 추가되었습니다.
* ``mine=true`` 는 현재 사용자가 담당자인 작업 또는 현재 사용자가 생성한 작업을 조회합니다.
* 현재 사용자가 생성했고 다른 담당자가 지정된 작업도 ``mine=true`` 결과에 포함됩니다.
* ``mine`` 은 ``assigneeMemberId`` 와 동시에 사용할 수 없습니다.
* 대시보드 화면에서 사용할 API를 모은 ``대시보드 API 조립 가이드`` 문서가 추가되었습니다.
