Integration Note

Added:: 작업 알림이 개인 알림 API와 WebSocket을 대상으로 추가되었습니다.
+
* ``TASK_CREATED``, ``TASK_DATE_CHANGED``, ``TASK_STATUS_CHANGED``, ``TASK_PROGRESS_CHANGED``, ``TASK_ASSIGNED``, ``TASK_UNASSIGNED`` 알림 유형이 추가되었습니다.
* 작업 알림의 ``subject.type`` 은 ``TASK`` 이며 작업 상세 이동 링크는 ``TASK_DETAIL`` 입니다.
* 작업 생성과 동시에 담당자로 지정된 사용자는 ``TASK_CREATED`` 이후 ``TASK_ASSIGNED`` 를 이어 받을 수 있습니다.
* 담당자 변경 시 직전 담당자에게는 ``TASK_UNASSIGNED``, 새 담당자에게는 ``TASK_ASSIGNED`` 가 전달됩니다.
* 상태, 담당자 변경은 각각의 작업 알림 유형으로 구분됩니다.
* ``TASK_DATE_CHANGED`` 와 ``TASK_PROGRESS_CHANGED`` 는 enum 값과 계약은 유지되지만 현재는 노이즈 감소 정책에 따라 알림 기록과 WebSocket payload를 생성하지 않습니다.

Changed:: 작업 알림의 ``changes`` 는 알림 유형별로 분리됩니다.
+
* ``TASK_ASSIGNED`` 와 ``TASK_UNASSIGNED`` 는 담당자 변경 요소를 기준으로 표시할 수 있습니다.
* ``TASK_STATUS_CHANGED`` 는 상태 변경을 기준으로 표시할 수 있습니다.
* 여러 변경이 동시에 발생해도 클라이언트는 ``type`` 을 1차 분기 기준으로 삼아 알림을 표시하면 됩니다.

Changed:: 알림 WebSocket과 목록 조회의 순서 안내가 보강되었습니다.
+
* 클라이언트는 ``/user/queue/notifications`` 로 내려오는 알림의 수신 순서를 유지해 표시할 수 있습니다.
* 같은 작업 이벤트에서 파생된 알림은 순차적으로 내려올 수 있으며 별도의 그룹 식별 값은 payload에 포함되지 않습니다.
* 재접속, 새로고침, 미접속 구간 복구는 계속 ``GET /v1/notifications`` 응답을 기준으로 보정합니다.

Docs:: 알림 WebSocket과 작업 알림 문서를 보강했습니다.
+
* ``작업 알림 WebSocket`` 문서가 추가되었습니다.
* ``알림 WebSocket`` 문서에 목록 API 보정 기준과 순서 처리 안내가 추가되었습니다.
* ``워크스페이스 멤버 WebSocket`` 문서에 연속 알림 처리 기준이 추가되었습니다.
* ``알림 목록 조회 API`` 문서의 정렬 설명이 보강되었습니다.

Compatibility:: 응답 구조는 유지됩니다.
+
* ``NotificationAlertResponse`` 필드 구조는 변경하지 않았습니다.
* 새 알림 유형과 ``TASK`` / ``TASK_DETAIL`` enum 값이 추가되었으므로 알 수 없는 enum 값에 대한 fallback 처리가 필요할 수 있습니다.
* WebSocket payload와 목록 API의 ``data.items[]`` 는 같은 알림 응답 구조를 사용합니다.
